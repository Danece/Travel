# Travel Mark — 手動測試腳本

> **版本**：1.0.0　**更新日期**：2026-04-30
>
> **執行環境**：實體 Android 裝置（API 26+）＋ iOS 裝置（iOS 15+）  
> **測試帳號**：準備一組有 Google Drive 存取權限的 Gmail 帳號  
> **前置條件**：資料庫乾淨（初次安裝或清除 App 資料後執行）
>
> 圖例：🔴 阻斷性缺陷（必修）　🟡 重要缺陷（上架前修）　🔵 輕微問題（可延後）

## 1. 標記 CRUD 完整流程

### 1.1 新增標記

- [ ] 點擊地標列表頁右下角「新增地標」FAB，導覽至新增頁面
- [ ] 標題欄位：輸入「東京鐵塔」，確認字元可正常輸入
- [ ] 國家下拉：選取「Japan」，確認選取後顯示正確
- [ ] 評分：點選 4 顆星，確認星星高亮狀態正確
- [ ] 日期：點擊日期欄位，DatePicker 彈出，選取非今日日期後確認顯示格式為 `yyyy/MM/dd`
- [ ] 心得：輸入多行文字（含中文、英文、emoji），確認換行正常
- [ ] 座標：點擊「在地圖上選點」，進入地圖選點頁後選取一個位置，確認緯度／經度回填至表單
- [ ] 點擊「儲存」，成功後自動返回列表頁
- [ ] 列表頁確認新標記出現於頂端（依建立時間降序排列）

### 1.2 查看詳情

- [ ] 點擊列表中的「東京鐵塔」卡片，進入詳情頁
- [ ] 確認標題、國家、日期、評分、座標、心得均正確顯示
- [ ] 確認導覽列標題正確顯示地標名稱
- [ ] 點擊返回按鈕，回到列表頁

### 1.3 編輯標記

- [ ] 進入詳情頁後點擊右上角編輯按鈕（鉛筆圖示）
- [ ] 修改標題為「東京鐵塔（修改版）」
- [ ] 修改評分為 5 顆星
- [ ] 修改心得文字
- [ ] 點擊「儲存」，確認返回詳情頁後所有修改已反映
- [ ] 回列表頁確認卡片標題已更新

### 1.4 刪除標記（列表頁左滑）

- [ ] 在列表頁對「東京鐵塔（修改版）」向左滑動，出現紅色刪除背景與圖示
- [ ] 放開後彈出確認 Dialog，確認標題文字正確
- [ ] 點擊「取消」，確認標記未被刪除，回到列表
- [ ] 再次左滑 → 點擊「刪除」，確認標記從列表消失
- [ ] 確認列表為空時顯示「尚無旅遊紀錄」空白頁

### 1.5 表單驗證

- [ ] 新增頁面：不填標題直接儲存，確認顯示「標題為必填」錯誤提示
- [ ] 不選國家直接儲存，確認顯示國家必填提示
- [ ] 不填座標直接儲存，確認顯示座標必填提示
- [ ] 填入無效緯度（如 `999`），確認顯示範圍錯誤提示
- [ ] 填入無效經度（如 `-999`），確認顯示範圍錯誤提示

## 2. 照片功能

### 2.1 拍照新增

- [ ] 進入詳情頁 → 點擊底部「新增照片」按鈕，彈出來源選擇面板
- [ ] 選擇「拍照」，相機開啟後拍攝一張照片
- [ ] 確認照片出現在頂部 PageView 中（縮圖可見）
- [ ] 確認頁碼標籤顯示「1 / 1」

### 2.2 從相簿選取（多選）

- [ ] 點擊「新增照片」→「從相簿選取」
- [ ] 一次選取 3 張照片
- [ ] 確認照片全部加入，頁碼更新為「1 / 4」（含先前 1 張）
- [ ] 左右滑動 PageView，確認每張照片顯示正常
- [ ] 確認照片儲存後（重新進入詳情頁）仍然存在

### 2.3 10 張上限

- [ ] 繼續新增照片直到共 10 張
- [ ] 第 10 張新增成功後，再次點擊「新增照片」
- [ ] 確認出現 Snackbar 提示「最多只能新增 10 張照片」
- [ ] 確認無法再繼續新增

### 2.4 刪除照片

- [ ] 在 PageView 長按其中一張照片
- [ ] 確認彈出確認 Dialog（含照片索引資訊）
- [ ] 點擊「取消」，確認照片未被刪除
- [ ] 再次長按 → 確認刪除，確認照片從 PageView 移除
- [ ] 確認頁碼標籤數字減少 1

### 2.5 缺失檔案顯示

- [ ] （進階）手動透過檔案管理器刪除 App Documents/photos/ 內一張照片
- [ ] 重新進入該標記詳情頁
- [ ] 確認缺失照片位置顯示 `broken_image` 灰色佔位圖示，不出現 crash 或紅色錯誤畫面

### 2.6 列表縮圖

- [ ] 回到列表頁，確認有照片的標記卡片左側顯示第一張照片縮圖（60×60）
- [ ] 確認無照片標記卡片顯示預設地點圖示佔位

## 3. 座標功能

### 3.1 地圖選點

- [ ] 新增/編輯頁面點擊「在地圖上選點」，確認地圖選點頁正確開啟
- [ ] 地圖預設定位至目前裝置位置（或台灣中心）
- [ ] 長按地圖任意位置，確認 PIN 圖示出現於點擊位置
- [ ] 確認底部顯示選取的緯度／經度（小數點 4 位以上）
- [ ] 點擊「確認」，確認座標回填至表單的緯度／經度欄位

### 3.2 GPS 定位按鈕

- [ ] 地圖選點頁點擊右上角定位按鈕
- [ ] 確認地圖移動並置中於目前 GPS 位置
- [ ] 確認 PIN 同步移至目前位置

### 3.3 座標顯示格式

- [ ] 詳情頁確認座標以 `緯度, 經度` 格式顯示（如 `35.6586, 139.7454`）
- [ ] 確認座標文字可長按複製

### 3.4 開啟 Google Maps

- [ ] 詳情頁點擊座標列
- [ ] 確認彈出提示或直接開啟 Google Maps（或系統地圖應用程式）
- [ ] 確認開啟的地圖定位正確

## 4. 搜尋與篩選

> **前置**：準備至少 10 筆涵蓋不同國家、評分、日期的測試資料。

### 4.1 標題模糊搜尋

- [ ] 搜尋欄輸入「東京」，確認只顯示標題含「東京」的結果
- [ ] 確認搜尋有 300ms debounce（快速輸入時不立刻送出請求）
- [ ] 清除搜尋文字（點 X 按鈕），確認列表恢復全部資料
- [ ] 輸入英文大寫「TOKYO」，確認 SQLite LIKE 不分大小寫有找到結果
- [ ] 輸入不存在的文字「ZZZZZ」，確認顯示空列表（非錯誤頁）

### 4.2 多國篩選

- [ ] 點擊「國家」篩選 Chip
- [ ] 確認 Dialog 內顯示現有資料的不重複國家清單（已排序）
- [ ] 勾選「Japan」→ 點確認，確認 Chip 變為選取狀態（藍色）
- [ ] 確認列表只顯示 Japan 的標記
- [ ] 再次點擊「國家」Chip，同時勾選「Japan」與「France」
- [ ] 確認列表顯示 Japan + France 的標記（聯集）
- [ ] 點擊「清除篩選」，確認 Chip 恢復未選取、列表顯示全部

### 4.3 最低評分篩選

- [ ] 點擊「評分」篩選 Chip，選取「3 星以上」
- [ ] 確認列表只顯示評分 ≥ 3 的標記
- [ ] 選取「5 星以上」，確認只顯示 5 星標記
- [ ] 若無 5 星標記，確認顯示空列表（非錯誤頁）

### 4.4 日期區間篩選

- [ ] 點擊「日期」篩選 Chip，開啟日期範圍選取器
- [ ] 選取一個有資料的月份區間，確認 Chip 顯示 `mm/dd – mm/dd` 格式
- [ ] 確認列表只顯示該區間內的標記
- [ ] 選取一個完全沒有資料的未來區間，確認顯示空列表

### 4.5 組合篩選

- [ ] 同時啟用：搜尋「Tower」+ 國家「Japan」+ 評分 ≥ 4
- [ ] 確認結果為三項條件的交集
- [ ] 清除全部篩選後，確認列表恢復全部資料
- [ ] 下拉刷新，確認搜尋條件被清除，全部資料重新載入

## 6. Excel 匯出入

### 6.1 匯出驗證

- [ ] 進入 Excel 頁面，點擊「匯出 Excel」按鈕
- [ ] 確認進度指示器出現（非空白畫面）
- [ ] 匯出成功後，確認出現分享面板（Share Sheet）
- [ ] 用 Excel 或 Google Sheets 開啟匯出的 .xlsx 檔案
- [ ] 確認第一列為標題行：標題、國家、緯度、經度、評分、日期、心得
- [ ] 確認資料行數與 App 內標記總數一致
- [ ] 確認日期欄格式為 `yyyy/MM/dd`
- [ ] 確認含中文、特殊字元的欄位無亂碼
- [ ] 確認評分欄為數字（1–5），非文字

### 6.2 下載範本

- [ ] 點擊「下載匯入範本」，確認下載 .xlsx 範本
- [ ] 開啟範本，確認第一列標題欄名稱與匯出格式一致
- [ ] 確認範本中有範例資料列（示範格式用）

### 6.3 匯入成功情境

- [ ] 在範本中填入 3 筆完整有效資料（不同國家、評分）
- [ ] 點擊「匯入 Excel」，選取填好的 .xlsx 檔案
- [ ] 確認匯入結果 Dialog 顯示「成功匯入：3 筆、跳過：0 列、驗證失敗：無」
- [ ] 確認 3 筆新資料出現於標記列表

### 6.4 匯入錯誤處理

- [ ] 準備包含以下錯誤的 .xlsx：第 3 列標題欄空白、第 5 列評分填 `abc`、第 7 列緯度超出範圍（999）
- [ ] 執行匯入，確認結果 Dialog 顯示可捲動的失敗清單
- [ ] 確認每筆失敗都顯示具體原因（如「第 3 列：標題不得為空」）
- [ ] 確認無錯誤的列仍正常匯入（部分成功）
- [ ] 選取非 .xlsx 格式檔案，確認出現錯誤提示或拒絕選取

### 6.5 空白列跳過

- [ ] 在 .xlsx 中插入幾列完全空白的列
- [ ] 匯入後確認結果 Dialog 的「跳過（空白列）」數量正確

### 7.5 還原備份

- [ ] 確認還原，確認 App 資料回到「備份 A」時的狀態

### 7.6 Google 登出

- [ ] 點擊帳號區塊的「登出」按鈕
- [ ] 確認登出後頁面回到未登入狀態
- [ ] 確認已登入過的資料未被清除（標記仍存在）

## 9. 邊界情境

### 9.1 無資料空白頁

- [ ] 清除所有標記資料（或使用全新安裝的 App）
- [ ] 地標列表頁確認顯示空白狀態：大地圖圖示 + 「尚無旅遊紀錄」文字 + 「立即新增」按鈕
- [ ] 點擊空白頁的「立即新增」按鈕，確認正確導覽至新增頁面
- [ ] 地圖頁面確認無標記時地圖正常顯示（無 crash）
- [ ] 搜尋頁輸入任何文字，確認顯示空列表而非錯誤頁

### 9.2 GPS 權限拒絕

- [ ] 在裝置設定中將 Travel Mark 的位置權限設為「拒絕」
- [ ] 開啟地圖頁，確認 App **不崩潰**，地圖正常顯示
- [ ] 確認「我的位置」按鈕不顯示或顯示為停用狀態
- [ ] 重新開啟 App，若系統彈出權限請求並點選「永久拒絕」
  - [ ] 確認顯示 AlertDialog：「需要位置權限」
  - [ ] 點擊「前往設定」，確認開啟系統 App 設定頁面
  - [ ] 點擊「取消」，確認 Dialog 關閉，App 正常運作
- [ ] 地圖選點頁在無位置權限時，確認地圖仍可手動拖曳選點

### 9.3 Google Drive 未登入保護

## 10. 效能測試

> **前置**：以下測試需先準備 100 筆測試標記資料。  
> 建議使用 Excel 匯入功能批次匯入，每筆涵蓋不同國家、評分、座標（盡量分散全球）。

### 10.1 列表頁捲動效能

- [ ] 地標列表有 100 筆資料時，確認初次載入時間 < 2 秒
- [ ] 快速上下捲動列表，確認無明顯卡頓（目標 60fps）
- [ ] 確認 Shimmer 骨架在資料載入前正常顯示（3 筆佔位卡片）
- [ ] 確認捲動時縮圖不出現頻繁的重新載入閃爍

### 10.2 地圖頁標記渲染效能

- [ ] 地圖頁載入 100 個標記，確認渲染時間合理（< 3 秒）
- [ ] 縮小 / 放大地圖，確認聚合計算不造成明顯 lag
- [ ] 快速拖曳地圖，確認標記跟隨不出現大量延遲
- [ ] 確認有照片的標記 icon（圓形縮圖）正常顯示，不出現 OOM

### 10.3 搜尋效能

- [ ] 有 100 筆資料時，輸入 1 個字元後（debounce 完成），確認搜尋結果 < 500ms 內呈現
- [ ] 連續快速輸入 10 個字元，確認僅最後一次觸發查詢（debounce 有效）

### 10.4 Excel 匯出效能

- [ ] 有 100 筆資料時執行匯出，確認完成時間 < 5 秒
- [ ] 確認匯出過程中 UI 不完全凍結（有進度指示）

### 10.5 記憶體使用

- [ ] 使用 Android Studio Profiler 或 Xcode Instruments 觀察
- [ ] 瀏覽有 100 筆資料的列表並捲動，確認記憶體無持續增長趨勢
- [ ] 進入地圖頁後，確認記憶體峰值在合理範圍（< 200MB）
- [ ] 退出地圖頁後，確認記憶體正確釋放（無明顯 leak）
